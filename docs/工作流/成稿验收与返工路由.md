# 成稿验收与返工路由

## 目标

检查阶段产物是否满足已确认的目标，并把问题准确打回到需要修改的环节。

## 检查维度

- 是否回应了任务目标和目标读者；
- 是否使用了可追溯的信息，且没有把推断写成事实；
- 结构是否完整，重点是否清楚；
- 是否满足格式、长度、交付渠道等明确约束；
- 是否还存在需要用户确认的方向性问题。

## 路由原则

| 发现的问题 | 返回环节 |
| --- | --- |
| 目标、受众或资料范围不清楚 | 分析 |
| 主题顺序、信息组织或论证关系有问题 | 结构规划 |
| 表达、衔接、篇幅或具体段落有问题 | 草稿生成 |
| 仅有事实待核验 | 查询或素材摄入 |

验收环节不直接代替前序环节重写全部内容；它输出“通过”或包含问题、证据和返工去向的返工单。
