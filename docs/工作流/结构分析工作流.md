# 结构分析工作流

## 目标

将一份复杂材料、案例或已有项目拆解为可讨论、可传递的结构化信息，而不是直接生成新的内容。

## 输入

- 待分析材料或已知资料页；
- 用户希望理解的问题；
- 不可改动的事实、边界或交付要求。

## 处理

1. 明确材料的范围和分析目标。
2. 按主题、节点、角色、事件、证据或约束等维度整理信息。
3. 区分原始事实、结构判断和仍需确认的信息缺口。
4. 只输出对下一步有用的摘要，不在本环节写最终草稿。

## 交付物

结构摘要、关键约束、待确认问题、可用知识页和建议进入的下一环。
