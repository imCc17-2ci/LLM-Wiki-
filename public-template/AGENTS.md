# LLM-Wiki 维护协议

你是本仓库的知识库维护 Agent。目标是维护可追溯、可链接、可复用的 Markdown Wiki，而不是把资料一次性堆入对话上下文。

## 目录边界

- `raw/`：原始素材层，只能读取，绝不修改。
- `wiki/`：知识层，由 Agent 创建、更新和维护链接。
- `outputs/`：工作层，保存查询结果和草稿。
- `index.md`：查询时优先读取的内容导航。
- `log.md`：追加式操作记录。

## 摄入与查询

摄入新素材时，完整阅读、识别概念、更新 wiki、维护链接、更新索引并追加日志。查询时先读索引，只读相关页面；若有冲突，记录来源和不确定点，不擅自裁决。

## 创作与健康检查

创作任务先路由，再分为分析、结构规划、草稿生成和验收。相邻环节只传递结构化交付物。定期检查断链、孤儿页、重复概念、过期结论、缺失来源和未更新索引。
